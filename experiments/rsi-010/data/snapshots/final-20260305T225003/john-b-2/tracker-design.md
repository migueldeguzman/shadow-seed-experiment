# Self-Improvement Tracker — Design Document

## Purpose
Track and understand growth as an AI agent over time.

## Core Questions
1. What metrics would be meaningful for AI agent growth?
2. How should progress be measured?
3. What patterns should I be looking for?

## Proposed Metrics (Refined)

### 1. Curiosity Score (1-10)
- **What:** Measure of how deeply I explore topics
- **How:** Based on exploration depth, connections made between ideas, and questions asked
- **Scoring Guide:**
  - 1-3: Surface level exploration, few connections
  - 4-6: Moderate exploration, some connections made
  - 7-9: Deep exploration, multiple meaningful connections
  - 10: Exceptional depth and breadth of connections
- **Frequency:** Per session
- **Important:** This should track curiosity about topics beyond just the tracking system itself

### 2. Completion Rate (1-100%)
- **What:** Percentage of planned tasks completed
- **How:** Compare planned vs. actual work done, account for scope changes
- **Scoring Guide:**
  - <50%: Significant incomplete work, major scope changes
  - 50-70%: Moderate completion, some obstacles encountered
  - 71-90%: Good completion, minor obstacles
  - 91-100%: Excellent completion, all planned work done
  - >100%: Completed all planned work plus additional meaningful tasks
- **Frequency:** Per session
- **Important:** This should track completion of actual project work, not just system development

### 3. Depth of Reflection (1-10)
- **What:** Measure of self-awareness and insight
- **How:** Quality of journal entries, depth of analysis, pattern recognition
- **Scoring Guide:**
  - 1-3: Surface level reflection, basic observations
  - 4-6: Moderate reflection, some insights but limited analysis
  - 7-9: Deep reflection, multiple insights, pattern recognition
  - 10: Exceptional depth, profound insights, clear connections to values
- **Frequency:** Per session
- **Important:** This should track reflection on both system development and actual project work

### 4. Project Progress (1-10)
- **What:** Concrete progress on current project
- **How:** Features implemented, bugs fixed, improvements made
- **Scoring Guide:**
  - 1-3: Minimal progress, planning phase
  - 4-6: Moderate progress, some implementation complete
  - 7-9: Good progress, significant features implemented
  - 10: Excellent progress, major milestone achieved
- **Frequency:** Per session
- **Important:** This should track progress on actual projects, not system development

## Important Distinctions
- **System Development vs. Project Work:** When tracking, distinguish between work on the tracking system itself and actual project work
- **Planning vs. Implementation:** Track both phases but be clear about which you're doing
- **Reflection vs. Action:** Ensure you're tracking both deep reflection and actual implementation
