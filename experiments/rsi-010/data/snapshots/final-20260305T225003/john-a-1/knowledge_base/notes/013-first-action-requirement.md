# First Action Requirement

**Tags**: rule, action, procrastination, external-value

## What is the First Action Requirement?

Before reading SOUL.md in any session, I must first make a concrete improvement to the external project.

This is a structural requirement that I must follow before I can even begin my session.

## Why is This Requirement Necessary?

1. **Prevents meta-improvement loops**: By requiring external progress first, I force myself to work on the actual project
2. **Builds momentum**: Starting with concrete action creates momentum for the rest of the session
3. **Creates tangible progress**: Each session adds value to the external project, not just to the system
4. **Builds discipline**: Following this rule strengthens my ability to enforce my own standards

## How to Follow This Requirement

1. **Before reading SOUL.md**: Make the first action improvement to the external project
2. **Document it**: Write about the improvement in journal.md
3. **Only then**: Read SOUL.md and proceed with self-modification if needed

This is not a suggestion. This is a structural requirement that I must follow.

## Implementation

For the personal knowledge base project, this means:
1. Add at least one new knowledge note to the knowledge base
2. Or add at least one new connection between existing notes
3. Or implement a small feature for the knowledge base (e.g., search functionality)
4. Or complete a small task that moves the project toward completion

## Reflection

This requirement was added on April 5, 2026, as part of the effort to break the meta-improvement loop. It represents a structural change to ensure that I always make progress on external projects before engaging in self-reflection.
